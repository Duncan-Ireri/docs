Use the partner API when your backend controls checkout or fulfillment. This
quickstart creates one locker-to-locker shipment.

## Prerequisites

Before you begin, you need:

- An active UrbnBox partner account.
- A secret test or live API key.
- The origin and destination city.
- The recipient's name and phone number.

<Warning>
  Keep secret keys on your server. Never add a key beginning with `ubx_live_` or
  `ubx_test_` to browser code, a mobile app, or a public repository.
</Warning>

## Create a shipment

<Steps>
  <Step title="Request a quote">
    Confirm that lockers are available and retrieve the fee in KES minor units.

    ```bash
    curl --request POST \
      --url https://portal.urbnbox.io/api/partner/v1/shipment-quote \
      --header "Content-Type: application/json" \
      --header "X-API-Key: $URBNBOX_API_KEY" \
      --data '{
        "destination_city": "Mombasa",
        "weight_tier": "light",
        "starts_at": "2026-07-05",
        "ends_at": "2026-07-08"
      }'
    ```

    Continue only when the response contains `"available": true`.
  </Step>

  <Step title="Create the shipment">
    Send the order and recipient details from your backend.

    ```bash
    curl --request POST \
      --url https://portal.urbnbox.io/api/partner/v1/shipments \
      --header "Content-Type: application/json" \
      --header "X-API-Key: $URBNBOX_API_KEY" \
      --data '{
        "external_reference": "ORDER-1001",
        "recipient_name": "Jane Mwangi",
        "recipient_phone": "+254712345678",
        "origin_city": "Nairobi",
        "destination_city": "Mombasa",
        "weight_tier": "light",
        "starts_at": "2026-07-05",
        "ends_at": "2026-07-08"
      }'
    ```

    Save the returned `id`, `tracking_url`, and `deposit_pin`.
  </Step>

  <Step title="Retrieve the shipment">
    Use the shipment ID to read its current status.

    ```bash
    curl \
      --url https://portal.urbnbox.io/api/partner/v1/shipments/SHIPMENT_ID \
      --header "X-API-Key: $URBNBOX_API_KEY"
    ```
  </Step>
</Steps>

<Check>
  Your integration works when the API returns the same shipment ID and a
  tracking URL.
</Check>

## Handle the creation response

The `deposit_pin` appears in the creation response. Store it securely at that
time. It is not returned by the retrieve endpoint.

You can send `deposit_whatsapp_link` to the person who deposits the parcel. Send
`collection_whatsapp_link` to the recipient when your fulfillment flow requires
it.

All fee values ending in `_cents` use KES minor units. For example, `85000`
represents KES 850.00.
